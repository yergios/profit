# Contributing guidelines

## Workflow rules

- No direct pushes to main branch. All changes will be through PR (Pull Requests) that need to have, at least, 1 approval
- Branch naming: Use issue label and number as prefix, e.g. `feature/123-add-cost-row`

## Formatting rules

- Indentation: 2 spaces
- File naming: kebab-cased
- Commit message format: [Conventional commits](https://www.conventionalcommits.org/en/v1.0.0/#summary)
